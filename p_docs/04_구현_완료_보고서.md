## 도메인 모델 (최종 확정)

### Server

```ts
interface Server {
  id: string;
  name: string;
  region: string; // 'kr-seoul' | 'us-east' | 'eu-west' | 'ap-tokyo'
  version: string;
  status: ServerStatus; // 'healthy' | 'degraded' | 'down'
  uptimeSeconds: number;
  lastHeartbeatAt: string; // ISO 8601
  playerCount: number;
  maxPlayers: number;
}
```

### MetricPoint

```ts
interface MetricPoint {
  timestamp: string;
  cpuUsage: number; // 0-100
  memoryUsage: number; // 0-100
  rttMs: number;
  errorRate: number; // 0-1
  playerCount: number;
}
```

### Alert

```ts
interface Alert {
  id: string;
  serverId: string;
  serverName: string;
  severity: "critical" | "warning" | "info";
  status: "open" | "acknowledged" | "resolved";
  title: string;
  description: string;
  ruleId: string;
  createdAt: string;
  resolvedAt?: string;
}
```

## 화면별 기능 요약

### 대시보드 (`/dashboard`)

- 서버 상태 요약 카드 4종 (전체 / healthy / degraded / down)
- 총 플레이어 수, 활성 서버 수, 평균 용량 수치 카드
- Recharts LineChart — 24시간 플레이어 추이
- 문제 서버 리스트 (degraded / down 우선 표시)
- 10초 자동 갱신 (TanStack Query 폴링)

### 서버 목록 (`/servers`)

- 3열 카드 그리드 (반응형)
- 상태 필터 (전체 / healthy / degraded / down)
- 지역 필터 (전체 / kr-seoul / us-east / eu-west / ap-tokyo)
- 카드 클릭 → 상세 이동

### 서버 상세 (`/servers/[id]`)

- 서버 메타 정보 카드 (업타임, 버전, heartbeat, 플레이어)
- 기간 선택 탭 (1h / 6h / 24h)
- 지표 차트 4종 (플레이어 / CPU / 메모리 / RTT)
- 관련 알림 리스트

### 알림 (`/alerts`)

- severity 필터 버튼 (전체 / critical / warning / info)
- status 필터 버튼 (전체 / open / acknowledged / resolved)
- 알림 행: 배지, 서버 링크, 제목, 상대 시간
- 상태 변경 버튼: "확인" (ack) → "해결" (resolved)
- 30초 자동 갱신
