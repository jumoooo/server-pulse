## Phase 4 — AI 분석 (기획서 Step 8)

> 핵심 차별화 기능. `/servers/[id]` 상세 페이지에서 서버 상태를 AI가 분석해 스트리밍으로 응답.

### 목표

운영자가 이상 서버의 `Analyze` 버튼을 클릭하면 현재 지표와 알림 데이터를 Claude에 전달하고, 분석 결과(원인 추정 + 권장 조치)를 실시간 스트리밍으로 표시한다.

### Task 4-1 · 분석 API Route

**파일**: `src/app/api/analyze/[id]/route.ts`

**흐름**

```
POST /api/analyze/[id]
  │
  ├─ params에서 serverId 추출
  ├─ mockData에서 서버 정보 + 최근 지표 + 활성 알림 수집
  ├─ 프롬프트 구성
  ├─ Anthropic API 스트리밍 호출
  └─ ReadableStream으로 텍스트 청크 응답
```

**프롬프트 설계**

```
System:
  당신은 게임 서버 SRE(Site Reliability Engineer) 전문가입니다.
  운영자에게 서버 이상 징후의 원인과 조치 방법을 한국어로 간결하게 안내합니다.
  반드시 아래 형식으로만 답변하세요:
  ## 현황 요약
  ## 원인 추정
  ## 권장 조치

User:
  서버: {name} ({region}, v{version})
  상태: {status} / 업타임: {uptime}

  최근 지표 (최신 3개 포인트):
  - CPU: {avg}% / 메모리: {avg}% / RTT: {avg}ms / 에러율: {avg}%
  - 플레이어: {current}/{max}

  활성 알림 ({count}개):
  - [{severity}] {title}: {description}
```

**응답 스트리밍 구현**

```ts
import Anthropic from "@anthropic-ai/sdk";

export async function POST(
  _: Request,
  { params }: { params: Promise<{ id: string }> }
) {
  const { id } = await params;
  // ... 데이터 수집, 프롬프트 구성

  const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });

  const stream = client.messages.stream({
    model: "claude-opus-4-6",
    max_tokens: 1024,
    system: SYSTEM_PROMPT,
    messages: [{ role: "user", content: userPrompt }],
  });

  const readable = new ReadableStream({
    async start(controller) {
      const enc = new TextEncoder();
      for await (const chunk of stream) {
        if (
          chunk.type === "content_block_delta" &&
          chunk.delta.type === "text_delta"
        ) {
          controller.enqueue(enc.encode(chunk.delta.text));
        }
      }
      controller.close();
    },
  });

  return new Response(readable, {
    headers: { "Content-Type": "text/plain; charset=utf-8" },
  });
}
```

**에러 처리**

| 케이스                     | 응답                                             |
| -------------------------- | ------------------------------------------------ |
| `ANTHROPIC_API_KEY` 미설정 | 500 + "API 키가 설정되지 않았습니다"             |
| 서버 ID 없음               | 404 + "서버를 찾을 수 없습니다"                  |
| Anthropic API 오류         | 502 + "분석 서비스에 일시적 오류가 발생했습니다" |

**스트리밍 구현 핵심**

```ts
const reader = response.body!.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  setText((prev) => prev + decoder.decode(value, { stream: true }));
}
```

**마크다운 렌더링**

응답 텍스트에 `##` 헤더가 포함되므로 간단한 파싱으로 섹션 구분:

- `## ` 로 시작하는 줄 → 강조 헤더 스타일
- 나머지 → 본문 텍스트
