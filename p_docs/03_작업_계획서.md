## Phase 0 — 기반 정비

### 목표

### Task 0-1 · 타입 정의 통일

**파일**: `src/types/server.ts`

| 항목      | 현재 코드               | 변경 후                                                                    |
| --------- | ----------------------- | -------------------------------------------------------------------------- |
| 서버 상태 | `"online" \| "offline"` | `"healthy" \| "degraded" \| "down"`                                        |
| 서버 지표 | `latencyMs`만           | `MetricPoint` 타입 추가 (CPU, 메모리, RTT, 에러율, 플레이어 수, timestamp) |
| 업타임    | 없음                    | `uptimeSeconds`, `lastHeartbeatAt` 필드 추가                               |
| 알림 상태 | 기본 구조               | `"open" \| "acknowledged" \| "resolved"` + `ruleId` 명시                   |

## Phase 1 — 데이터 레이어

> Phase 0 완료 후 착수. UI와 데이터를 분리해서 화면 작업을 빠르게 만든다.

### 목표

TanStack Query 훅과 Zustand store를 완성해 Phase 2 UI가 바로 꽂아 쓸 수 있는 상태를 만든다.

### Task 1-1 · TanStack Query 훅

**파일**: `src/hooks/`

| 훅                            | 역할                         | 폴링 |
| ----------------------------- | ---------------------------- | ---- |
| `useServers()`                | 전체 서버 목록               | 10초 |
| `useServerDetail(id)`         | 단일 서버 상세               | 10초 |
| `useServerMetrics(id, range)` | 서버 지표 시계열             | 30초 |
| `useAlerts(filters?)`         | 알림 목록 + 필터             | 30초 |
| `useAlertMutation()`          | 알림 상태 변경 (ack/resolve) | —    |

각 훅의 반환 형태 통일:

```ts
{
  (data, isLoading, isError, error, refetch);
}
```

```
커맨드: /feature TanStack Query 훅 5개 구현
```

### Task 1-2 · Zustand store 확장
