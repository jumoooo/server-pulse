> 목적: ServerPulse 프로젝트에 최적화된 Claude Code 에이전트/스킬 전체 셋업 과정 정리

## 1. 셋업 배경

기획서(`00_프로젝트_기획서_산출물.md`) 분석을 통해 아래 두 가지 갭을 발견했다.

| 항목                  | 문제                                                                           |
| --------------------- | ------------------------------------------------------------------------------ |
| **Step 8 AI 분석**    | Anthropic API 스트리밍 기능 구현에 필요한 `claude-api` 스킬 미설치             |
| **코드 리뷰**         | `agentCritic` 에이전트가 있지만 슬래시 커맨드(`/review`)가 없어 직접 호출 불가 |
| **목 데이터 생성**    | 기획서 전반에 걸쳐 Mock 데이터 우선 개발이 명시되어 있으나 전용 스킬 없음      |
| **에이전트 MCP 연결** | `planner`, `feature`, `api`, `ui`, `test` 에이전트에 MCP tools 미지정          |

## 2. 스킬 셋업

### 2-1. 외부 스킬 설치

```
anthropics/skills@claude-api  (5,100+ 설치 · 공식 Anthropic)
```

**설치 위치**: `.agents/skills/claude-api/` → `.claude/skills/claude-api/` 심링크
**설치 명령**: `npx skills add anthropics/skills@claude-api -y`

**사용 시점** (기획서 Step 8):

- `/servers/[id]` 상세 페이지 `Analyze` 버튼
- 서버 상태 데이터 → Anthropic API 전달 → 스트리밍 응답 표시
- TypeScript용 `.stream()` + `.finalMessage()` 패턴 제공

---

### 2-2. 프로젝트 신규 스킬 생성

#### `/review` — 코드 품질 검토

```
.claude/skills/review/SKILL.md
```

- `agentCritic` 에이전트를 슬래시 커맨드로 직접 호출
- 파일 경로 지정 → Quick Review / 기능명 지정 → Feature Review / 인수 없음 → 자동 감지
- 결과: 🔴 Critical / 🟡 Warning / 🔵 Info / 🟢 Good + 전체 등급 A/B/C

#### `/mock` — 목 데이터 생성

```
.claude/skills/mock/SKILL.md
```

- ServerPulse 3대 도메인 타입 기반 (`Server`, `MetricPoint`, `Alert`)
- 인수에 따라 자동 분기:

  | 키워드                     | 생성 결과                               |
  | -------------------------- | --------------------------------------- |
  | `fixture`, `test`          | `src/__tests__/fixtures/` 테스트 픽스처 |
  | `api`, `route` 또는 미지정 | `src/app/api/*/route.ts` Mock 응답      |
  | `seed`, `dev`              | 로컬 개발용 다양한 상태 시드            |

- 현실적 데이터 기준: 지역(`kr-seoul`, `us-east`, `eu-west`), status 비율(healthy 70% / degraded 20% / down 10%) 등

## 3. 에이전트 업데이트

### 변경 사유

에이전트 정의 파일에 `tools` 필드가 없으면 기본 도구만 사용되어 **MCP 서버가 연결되지 않는다**. `CLAUDE.md`에 명시된 MCP 연결을 실제로 활성화하기 위해 각 에이전트에 tools를 명시했다.

### 업데이트 내역

| 에이전트       | 변경 내용                                            | 이유                                         |
| -------------- | ---------------------------------------------------- | -------------------------------------------- |
| `planner`      | context7 MCP 추가                                    | 아키텍처 결정 시 Next.js 15 최신 패턴 조회   |
| `feature`      | context7 MCP 명시                                    | 구현 전 공식 패턴 검증                       |
| `api`          | context7 MCP 명시                                    | Route Handler / Server Action 최신 패턴 확인 |
| `ui`           | context7 MCP 명시                                    | Tailwind v4 클래스, 컴포넌트 패턴 확인       |
| `test`         | playwright MCP 명시                                  | E2E 테스트 시 실제 셀렉터/UI 확인            |
| `orchestrator` | 레지스트리에 슬래시 커맨드 컬럼 추가, `/review` 반영 | agentCritic 직접 호출 경로 명확화            |

### 에이전트별 최종 tools

```yaml
planner: Read, Grep, Glob, WebFetch, WebSearch, context7(×2)
feature: Read, Grep, Glob, Write, Edit, Bash, WebFetch, context7(×2)
api: Read, Grep, Glob, Write, Edit, Bash, WebFetch, context7(×2)
ui: Read, Grep, Glob, Write, Edit, Bash, WebFetch, context7(×2)
test: Read, Grep, Glob, Write, Edit, Bash, WebFetch, playwright(×5)
commit: Bash, Read  (변경 없음 — 경량 haiku 모델 유지)
agentCritic: Read, Grep, Glob, WebFetch, context7(×2)  (변경 없음)
orchestrator: Agent, Read, Glob, Grep, WebFetch, sequential-thinking  (변경 없음)
```

## 4. 최종 에이전트 시스템 구조

```
사용자 요청
    │
    ├── 단순 작업 ──────────────────────────────────────────────────────┐
    │   /feature | /ui | /api | /test | /commit | /review | /mock      │
    │                                                                    │
    └── 복잡한 작업 ─── /orchestrate ────────────────────────────────────┤
                              │                                         │
                    Intent 분류 + 복잡도 평가                           │
                              │                                         │
              ┌───────────────┼───────────────────┐                    │
              │               │                   │                    │
        [Sequential]    [Parallel]          [Review Loop]              │
        planner         ui + api            agentCritic (/review)      │
            ↓           (병렬)                  ↓                      │
        feature             ↓               feature (수정)             │
            ↓           feature                                        │
          test          (통합)                                          │
            ↓                                                           │
         commit ◀──────────────────────────────────────────────────────┘
```

### 워크플로우 패턴 요약

| 상황                  | 권장 커맨드                            |
| --------------------- | -------------------------------------- |
| 기능 개발 (단순)      | `/feature [기능명]`                    |
| 기능 개발 (복잡)      | `/orchestrate [요청]`                  |
| UI 컴포넌트           | `/ui [컴포넌트명]`                     |
| API 엔드포인트        | `/api [리소스명]`                      |
| 테스트 작성           | `/test [파일경로]`                     |
| 코드 리뷰             | `/review [파일경로]`                   |
| 목 데이터 필요        | `/mock [server\|metrics\|alerts\|all]` |
| AI 분석 구현 (Step 8) | `/claude-api`                          |
| 커밋/푸시             | `/commit`                              |
| 프로젝트 파악         | `/discover`                            |
