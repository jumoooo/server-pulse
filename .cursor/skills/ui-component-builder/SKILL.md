# UI Component Builder Skills

## Language Separation
**Internal (Agent reads)**: English. **User-facing**: Korean.

## Overview

Core functions: component type analysis, pattern research (Indexing & Docs first), component design, code generation with quality checks, responsive design, error handling.

## Skills

### 1. Analyze Component Requirements
Parse type (button/card/list/form/custom), properties, styling (Material/Cupertino/Custom), state needs.

### 2. Research Official Patterns
**Priority**: Indexing & Docs (Flutter_docs) → MCP Context7 → Codebase Search.

### 3. Check Project Structure
Use deep discovery report in `.cursor/docs/deep-discovery/` for existing patterns.

### 4. Design Component Architecture
StatelessWidget vs StatefulWidget, proper separation, const usage.

### 5. Generate Component Code
Clear structure, Korean comments, type safety, error handling, responsive design.

## Quality Standards

- Follow Flutter best practices
- Reusable, composable components
- Korean comments for complex logic
- Consistent with project patterns
