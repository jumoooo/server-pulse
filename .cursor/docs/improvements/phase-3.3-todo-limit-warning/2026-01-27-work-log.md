## 🔄 동작 방식

### 1. 할일 추가 시 체크

1. 사용자가 할일을 추가하려고 시도
2. `TodoProvider.addTodo()` 호출
3. 할일 개수 제한 체크:
   - 현재 개수 >= 제한 개수 → 예외 발생, 에러 스트림에 알림
   - 현재 개수 < 제한 개수 → 정상 진행
4. 경고 임계값 체크:
   - 현재 개수 >= 경고 임계값 (80개) → 경고 스트림에 알림
   - 경고 스트림 구독자(UI)가 다이얼로그 표시

### 2. 경고 다이얼로그 표시

- 할일 개수가 80개에 도달하면 자동으로 경고 다이얼로그 표시
- 사용자가 "확인" 버튼을 누르면 다이얼로그 닫힘
- 할일 추가는 계속 가능 (제한에 도달하기 전까지)

### 3. 제한 초과 시

- 할일 개수가 100개에 도달하면 추가 불가
- 에러 메시지 표시: "할일 개수 제한(100개)에 도달했습니다."
- 사용자는 일부 할일을 삭제한 후 다시 시도해야 함
