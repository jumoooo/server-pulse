### 3. featureImplementation
**역할**: 실제 코드 구현 및 버그 수정

**수행 작업**:

#### Phase 3.1 일괄 선택/삭제 기능 구현

**1. 상태 관리 추가** (`lib/widgets/calendar_widget.dart`)
- `_isSelectionMode`: 선택 모드 활성화 여부
- `_selectedTodoIds`: 선택된 할일 ID 목록 (Set<String>)
- `_toggleSelectionMode()`: 선택 모드 토글
- `_toggleTodoSelection()`: 개별 할일 선택/해제
- `_toggleSelectAll()`: 전체 선택/해제

**2. 일괄 삭제 메서드 추가** (`lib/providers/todo_provider.dart`)
- `deleteTodos(List<String> ids)`: 여러 할일 일괄 삭제
- 롤백 지원 및 undo 스택 통합
- 에러 처리 및 예외 처리

**3. UI 구현** (`lib/widgets/calendar_widget.dart`)
- 선택 모드 진입 버튼 (헤더)
- 선택 모드 헤더 (선택 개수, 전체 선택, 종료 버튼)
- 할일 아이템 체크박스 (선택 모드일 때만)
- 선택된 아이템 배경색 변경
- 일괄 삭제 버튼 및 확인 다이얼로그
- 삭제 후 SnackBar (되돌리기 버튼 포함)
- `InkWell` onTap 동작 수정 (선택 모드 vs 일반 모드)

**4. 테스트 작성** (`test/providers/todo_provider_test.dart`)
- 일괄 삭제 성공 테스트
- 일괄 삭제 후 undo 스택 확인 테스트
- 빈 리스트 삭제 테스트
- 존재하지 않는 ID 삭제 테스트

#### 정렬 기능 버그 수정

**1. 등록순 정렬 로직 추가** (`lib/providers/todo_provider.dart`)
- `byCreation` 케이스에 ID 기반 역순 정렬 추가
- 최신 등록 항목이 위에 표시되도록 수정

**2. 하드코딩된 정렬 제거** (`lib/widgets/calendar_widget.dart`)
- `currentTodos`와 `neighborTodos`에서 하드코딩된 우선순위 정렬 제거
- `getTodosByDate()`가 반환하는 정렬된 목록을 그대로 사용
- 사용되지 않는 `_priorityRank()` 함수 제거

## 🐛 발견 및 수정된 버그

### 1. 정렬 기능 버그
**문제**: 
- 등록순 정렬이 작동하지 않음 (정렬 로직 없음)
- 이름순과 등록순이 같은 순서로 표시됨
- 하드코딩된 우선순위 정렬이 사용자 선택을 무시함

**원인**:
- `byCreation` 케이스에서 정렬 로직이 없음 (`break;`만 있음)
- `calendar_widget.dart`에서 `getTodosByDate()` 결과를 다시 우선순위로 정렬함

**해결**:
- `byCreation` 케이스에 ID 기반 역순 정렬 추가
- 하드코딩된 정렬 제거
